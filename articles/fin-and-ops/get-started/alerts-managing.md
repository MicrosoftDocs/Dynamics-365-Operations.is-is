### <a name="frequency-for-change-based-events"></a><span data-ttu-id="1426e-116">Tíðni breytingabyggðra atvika</span><span class="sxs-lookup"><span data-stu-id="1426e-116">Frequency for change-based events</span></span>

<span data-ttu-id="1426e-120">Á hinn bóginn, runuvinnsla sem endurtekur sig sjaldnar, og er tímasett þegar álag á kerfinu er lítið, getur hjálpað til við að bæta afköst kerfisins.</span><span class="sxs-lookup"><span data-stu-id="1426e-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="1426e-121">Hins vegar getur lág tíðni á runuvinnslu hugsanlega ekki mætt kröfum notenda um tímanlegar viðvaranir.</span><span class="sxs-lookup"><span data-stu-id="1426e-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="1426e-122">Þ.a.l. þegar þú setur upp tíðni runuvinnslu fyrir breytingabyggð atvik skaltu íhuga jafnvægið á milli tímasetninga á viðvörunum og afköstum kerfisins.</span><span class="sxs-lookup"><span data-stu-id="1426e-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="1426e-123">Þetta mun skipta meira máli eftir því sem að notendum fjölgar sem búa til viðvörunarreglur.</span><span class="sxs-lookup"><span data-stu-id="1426e-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="1426e-124">Tíðnin hefur ekki áhrif á fjölda atvika sem þarf að vinna úr.</span><span class="sxs-lookup"><span data-stu-id="1426e-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="1426e-125">Hins vegar, ef fleiri notendur búa til reglur, verða fleiri athuganir gerðar.</span><span class="sxs-lookup"><span data-stu-id="1426e-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="1426e-126">Þessi gerð af gagnaskiptum getur haft áhrif á afköst kerfisins.</span><span class="sxs-lookup"><span data-stu-id="1426e-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="1426e-127">Hætturnar á lágri runutíðni</span><span class="sxs-lookup"><span data-stu-id="1426e-127">The risks of low batch frequency</span></span>

<span data-ttu-id="1426e-130">Til dæmis er viðvörunarregla sett á til að kveikja á viðvörun þegar atvikið er **breytingar á tengilið viðskiptavinar** og skilyrðin eru **viðskiptavinur = BB**.</span><span class="sxs-lookup"><span data-stu-id="1426e-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="1426e-131">Með öðrum orðum, þegar tengilið viðskiptavinar er breytt fyrir viðskiptavin BB, er atvikið skráð.</span><span class="sxs-lookup"><span data-stu-id="1426e-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="1426e-132">Hins vegar er runuvinnslukerfið sett upp þannig að runuvinnsla á sér stað sjaldnar en gagnaskráning.</span><span class="sxs-lookup"><span data-stu-id="1426e-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="1426e-133">Ef nafn viðskiptavinar er breytt úr **BB** í **AA** áður en unnið er úr atvikinu, samsvara gögnin í gagnagrunninum ekki lengur skilyrðunum í reglunni, **viðskiptavinur = BB**.</span><span class="sxs-lookup"><span data-stu-id="1426e-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="1426e-134">Þannig að þegar loksins er unnið úr atvikinu er engin viðvörun sett af stað.</span><span class="sxs-lookup"><span data-stu-id="1426e-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
