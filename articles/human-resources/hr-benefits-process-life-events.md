# <a name="process-life-events"></a><span data-ttu-id="ef28a-103">Vinna úr viðburðum</span><span class="sxs-lookup"><span data-stu-id="ef28a-103">Process life events</span></span>

[!include [banner](includes/preview-feature.md)]

<span data-ttu-id="ef28a-104">Í líftíma starfsmannsins í Microsoft Dynamics 365 Human Resources, getur hver starfsmaður lent í ýmsum breytingum á atburði í lífinu.</span><span class="sxs-lookup"><span data-stu-id="ef28a-104">During the employee lifecycle in Microsoft Dynamics 365 Human Resources, each employee may encounter various life event changes.</span></span> <span data-ttu-id="ef28a-105">Til dæmis hjónaband, breyting á atvinnu eða breyting á skjólstæðingi/rétthafi.</span><span class="sxs-lookup"><span data-stu-id="ef28a-105">For example, marriage, change in employment, or dependent/beneficiary change.</span></span> <span data-ttu-id="ef28a-106">Til að nota atburði í lífinu verður þú að virkja lífatburði á formi hagur breytur, setja upp gerðir atburða fyrir líf og setja upp val á atburði fyrir áætlunartegundir.</span><span class="sxs-lookup"><span data-stu-id="ef28a-106">To use life events, you must enable life events in the benefits parameters form, set up life event types, and set up life event options for plan types.</span></span>

<span data-ttu-id="ef28a-107">Áður en þú getur afgreitt atburði í lífinu verður þú að hafa þegar rekið opna skráningu amk einu sinni á ráðningartíma.</span><span class="sxs-lookup"><span data-stu-id="ef28a-107">Before you can process life events, you must have already run open enrollment at least once during a hiring time frame.</span></span> <span data-ttu-id="ef28a-108">Í Bandaríkjunum er opin innritun venjulega einu sinni á ári.</span><span class="sxs-lookup"><span data-stu-id="ef28a-108">In the United States, open enrollment is typically once per year.</span></span> <span data-ttu-id="ef28a-109">Utan Bandaríkjanna kann að vera opin innritun við ráðningu.</span><span class="sxs-lookup"><span data-stu-id="ef28a-109">Outside the United States, open enrollment may be run at the time of hire.</span></span> <span data-ttu-id="ef28a-110">Starfsmaður þarf ekki að velja bótaáætlun til að lífatburðir verði afgreiddir, en þeir þurfa að hafa verið með í opinni skráningarvinnslu.</span><span class="sxs-lookup"><span data-stu-id="ef28a-110">A worker does not need to select a benefit plan in order for life events to be processed, but they need to have been included in open enrollment processing.</span></span> 

<span data-ttu-id="ef28a-111">Notaðu vinnslu lífsviðburða þegar þú ert með starfsmenn sem eru með atburði í lífinu sem eiga sér stað á framtíðardegi.</span><span class="sxs-lookup"><span data-stu-id="ef28a-111">Use life event processing when you have workers who have life events that take place on a future date.</span></span> <span data-ttu-id="ef28a-112">Þessi atburður vinnur alla atburði í lífinu sem ekki hafa verið afgreiddir (eins og atburðir í framtíðinni, eða lífatburðir sem hafa verið bættir við sem eru ekki sérstakir fyrir einn starfsmann - eitt dæmi er nýr ávinningur).</span><span class="sxs-lookup"><span data-stu-id="ef28a-112">This event will process all life events that have not been processed (like future life events, or life events that have been added that are not specific to any one worker – one example is a new benefit).</span></span> <span data-ttu-id="ef28a-113">Atburðir í rauntíma eru faldir.</span><span class="sxs-lookup"><span data-stu-id="ef28a-113">Real time life events are hidden.</span></span>

<span data-ttu-id="ef28a-114">Til dæmis, ef í dag er 1. febrúar, og þann 14. febrúar, áætlar starfsmaður Joe Smith að breyta lögaðilum, ef þú rekur vinnslu atburða fyrir 15. febrúar, vinnur kerfið alla atburði til 15. febrúar.</span><span class="sxs-lookup"><span data-stu-id="ef28a-114">For example, if today is February 1, and on February 14 worker Joe Smith is scheduled to change legal entities, if you run life event processing for February 15, the system processes all events up until February 15.</span></span>
